# 애드센스 승인 확률 극대화 — 심층 자료 (2026)

> 기본 설계(`00_애드센스_승인설계.md`)에 더해, **거절을 부르는 진짜 원인과 합격을 가르는 고급 요인**을 정리.
> 핵심 결론: **우리 계산기 = "독자적 데이터" = 거절 1위 "저가치 콘텐츠"를 구조적으로 이기는 무기.**

---

## 1. 심사관이 실제로 보는 것 = E-E-A-T (Trust가 최우선)

| 요소 | 의미 | 우리 적용 |
|---|---|---|
| **Experience** | 저자의 1차 경험 | 글에 "직접 계산해보니/실제 사례" 삽입 |
| **Expertise** | 진짜 아는가 | 표준 공식·출처 명시, 정확성 검증 |
| **Authoritativeness** | 권위 신호 | About에 운영주체·미션, 일관된 니치 |
| **Trustworthiness** ⭐ | 가장 중요 | 면책·출처·연락처·HTTPS·투명성 |

→ 각 글에 **저자 표기(byline) + 출처 링크 + "마지막 업데이트" 날짜**를 넣으면 Trust 신호가 크게 오른다.

---

## 2. ⭐ "30% 규칙" + Information Gain — 우리의 최대 강점

**거절의 진짜 본질 = 정보 중복(Information Redundancy).**
- 2026 구글 AI 크롤러는 **"Information Gain"**(이 페이지에만 있는 정보)을 찾는다.
- 인사이더 지표 **"30% 규칙"**: 콘텐츠의 **최소 30%가 인터넷 어디에도 없는 정보**여야 합격.
- "블로그 시작하는 법" 같은 10만 개 사이트와 똑같은 글 = 자동 저가치.

**해법 = 독자적 데이터(Proprietary Data) 주입.**
> **계산기가 바로 독자적 데이터다.** 사용자가 입력→계산→고유 결과를 받는 인터랙티브 도구는 "다른 사이트의 미러"가 아니라는 결정적 증거다. 단순 텍스트 블로그가 30% 규칙에 막힐 때, **계산기 사이트는 도구 자체로 이를 통과**한다. → 툴 사이트의 구조적 우위.

여기에 **주별 평균값·실측 데이터·비교표** 같은 고유 데이터 포인트를 더하면 Information Gain이 폭발한다.

---

## 3. 저가치 콘텐츠 거절 — 정확한 5대 원인 & 해결

| 원인 | 기준 | 해결 |
|---|---|---|
| **얇은 콘텐츠** | 600단어 미만, 표면적 | **1,500단어+**(2026 스위트스팟), 깊이 있게 |
| **고유정보 부재** | 10만 사이트와 동일 | 독자 데이터·계산기·고유 예시 |
| **검색의도 불일치** | 체크리스트 검색에 역사 3000자 | 원하는 답을 빠르게 먼저 |
| **복사/파생 콘텐츠** | 스크랩·재가공만 | 원본 분석·관점 추가 |
| **저품질 AI 콘텐츠** | 95% AI 양산 | **50:50 이상 사람 가치 추가**(검토·사실확인·경험) |

**글 작성 공식**: 독자 고통 공감 → 핵심 개념 정의 → 이유 → 단계별 해결 + before/after 비교. 각 1,500단어+.

---

## 4. 합격을 가르는 고급 요인 (대부분이 놓치는 것)

1. **GSC "수익 창출(Monetization)" 탭으로 신청** ⭐ — 독립 애드센스 포털보다, **Google Search Console의 Monetization 탭**을 통해 신청하면 사이트 성과 데이터가 연결돼 유리(인사이더 팁).
2. **신청 전 색인(indexed) 필수** — 글이 **구글에 실제 색인**된 뒤 신청. GSC에서 `site:yourdomain.com`로 확인.
3. **ads.txt 설정** — 루트에 `ads.txt` 배치(이 폴더에 템플릿 생성). 누락 시 "저품질 사이트" 신호.
4. **깨진 페이지·링크 0** — 기술 결함은 사이트 전체를 저품질로 보이게 함.
5. **모바일 우선 심사** — 심사관은 모바일 뷰부터 본다. 반응형·빠른 로딩 필수.
6. **도메인 3~6개월 연령** — 장기 프로젝트 신호. 도메인 먼저 사두고 콘텐츠 쌓기.
7. **일관된 단일 니치** — 잡다한 주제 X. "금융 계산기" 한 우물.
8. **논리적 내비/메뉴** — 지저분한 사이드바·복잡한 메뉴가 거절 유발.

---

## 5. 거절당했을 때 — 재신청 전략

- **2~3주 기다린 뒤** 재신청 (즉시 재신청 = 거의 같은 결과)
- **기존 글을 고쳐라**, 새 글만 더 얹지 마라
- **코스메틱 수정(테마·폰트) 금지** — 거절 사유와 무관
- 저가치 콘텐츠 거절은 **페널티 아님·영구밴 아님** — 가장 고치기 쉬운 거절. 본질(콘텐츠 깊이·고유성)만 해결하면 통과

---

## 6. 우리 프로젝트 — 승인 확률 강화 액션 (구체)

기존 골격에 더할 것:

- [ ] **각 지원글 1,500단어+**, byline·출처·업데이트 날짜 추가(E-E-A-T)
- [ ] **계산기 페이지에 고유 데이터** 추가(예: 주별 평균 모기지율 표) → Information Gain
- [ ] **`ads.txt`** 루트 배치(승인 후 publisher ID 기입)
- [ ] 사이트맵 생성 → **GSC 제출 → 색인 확인** 후 신청
- [ ] **GSC Monetization 탭**으로 신청
- [ ] 도메인 3~6개월 묵힌 뒤 신청(그동안 글·트래픽 축적)
- [ ] 글마다 관련 계산기 **내부링크**(체류·내비 강화)
- [ ] 모바일 Lighthouse 90+ 확인

---

## 7. 승인 확률 체크 — 통과 가능성 자가진단

| 신호 | 가중치 |
|---|---|
| 필수 4페이지 + 푸터 링크 | ✅ 이미 충족 |
| 인터랙티브 계산기(독자 데이터=30% 규칙) | ✅ 강점 |
| 지원글 15편 × 1,500단어, 사람 검토 | ⏳ 채워야 함 |
| 색인 완료 + ads.txt + 사이트맵 | ⏳ 배포 후 |
| 도메인 3~6개월 + 약간의 트래픽 | ⏳ 시간 필요 |

→ **위 ⏳ 3개만 채우면 합격권.** 골격·필수페이지·도구는 이미 완비.
